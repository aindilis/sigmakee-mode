A DAML Mode for XEmacs and GNU Emacs
This is the beta release of the DAML mode for the Emacs editor. For more information about the DAML language and the semantic web see The DAML Program Web Site.

Anyone who has tried to write DAML ontologies in a text editor will recognize that it requires a lot of typing, and that reading and moving around in such files can be tedious. This is (hopefully) a step in the direction of addressing those problems. DAML-EMACS is a "mode" for use with the EMACS text editor, not unlike the modes for C++, Java, LateX, LISP and HTML... that existed previously. It provides automatic fontification to make the code easier to read, commands to insert new forms to be filled out, and keyboard commands for moving about the file in logical DAML/RDF or XML units.

Software Requirements

This current release is was developed for XEmacs version 21.1, but is being adapted to also work with GNU Emacs (versions from 19.28 up). It has been tested with GNU version 20, and works acceptably there, as long as you have a recent version of the W3 package for Emacs.

We recommend using XEmacs with DAML mode over GNU Emacs. It is faster and the fonts are much better. The W3 mode version mentioned above works with XEmacs as well, although the version available from http://www.xemacs.org is also fine.

If you need/want to install XEmacs on a windows platform, there are two installation sets for version 21.4.3 available at http://www.xemacs.org, and they are also downloadable from here:

  • A version for use in conjunction with Cygwin (a unix-like shell that runs in Windows): Download non-CYGWIN XEmacs..
  • A version for use without Cygwin: Download CYGWIN XEmacs.

When you install either of these, and try to run it for the first time, it will try to create an init file based on your prior GNU ~/.emacs file if one exits. You can avoid some confusion here by first creating the ~/.xemacs/init.el file with nothing in it, and then moving things over from your .emacs initialization file gradually.

DAML mode the emacs W3 package to access web pages. The W3 mode version 4.0 installation tar from the University of Indiana, but you can also just Download W3 mode from here.

After unpacking it, see the file INSTALL for a installation on UNIX or the README.NT for windows installation.

Installing DAML Mode

The beta release source code for DAML mode is downloadable from this tar file: daml-emacs-beta.tar

  1. Unpack the files into a directory named daml/ somewhere, such as ~/emacs/daml
  2. Edit your ~/.emacs (GNU or UNIX XEmacs) or ~/.xemacs/init.el (windows XEmacs) initialization file so that the load-path includes the ~/emacs/daml/ (or wherever you put them) directory, and the directory containing the w3 package (for xemacs this is already there)..
  3. (optional) Compile the .el files in this directory. This seems to make the most difference for GNU emacs. To do it, in emacs do the following commands:
    1. M-x cd (at the prompt enter the code directory, e.g. "~/emacs/daml")
    2. M-x load-file (at the prompt enter "docomp")
To fix the load path (2 above), add the following two lines to the initialization file:

(setq load-path
(append '("<path to the emacs code directory>/daml/" "<path to w3 directory>")
load-path))
(load "damlite")

The w3 path is needed only if installing it yourself for use with GNU Emacs.

Using DAML Mode

See the README with the distribution for more details.

Once daml mode is installed, just open or create a file with extension .daml or edit any file and do

M-x DAML-MODE

If you open a non-existent file, like "newfile.daml", some default header information plus the rdf:RDF and /rdf:RDF tags are inserted. See 'daml-helper-new-buffer-template' in the file damlite.el to change what this looks like.

To get full use DAML-MODE, you will have to ascertain that the W3 package is installed on your machine, and that the load path makes it accessible. If it is not installed, you will not be able to use M-. to find DAML definitions on the web. To see if it is installed do

M-x locate-library (ENTER) w3 (ENTER)

If already installed, you will probably find it as a subdirectory under the XEmacs installation directory called
...mumble.../lisp/w3/

If you can't find it, you will have to install it. A copy of the tar'd (and gzipped) source for this package is available from http://www.xemacs.org/Download/optLibs.html or you can just Download W3 Mode from here.
